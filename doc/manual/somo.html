SOMO - Solution Modeler (Bead Modeler for NMR and X-ray structures):

The SOMO (SOlution MOdeller) module of UltraScan is a bead modelling utility that was originally developed by the Rocco and Byron labs, respectively at the Istituto Nazionale per la Ricerca sul Cancro (IST, Genova, Italy) and at the University of Glasgow (Glasgow, Scotland, UK). It is based on a reduced representation of a biomacromolecule, starting from its atomic coordinates, as a set of non-overlapping beads of different radii, from which the hydrodynamic properties can be calculated using the Garcia de la Torre-Bloomfield rigid-body approach (García de la Torre and Bloomfield, Q. Rev. Biophys. 14:81-139, 1981). The reduced representation is afforded by grouping together atoms and substituting them with a bead of the same volume, appropriately positioned. Importantly, the volume of the water of hydration theoretically bound to each group of atoms can be then added to each bead. The overlaps between the beads are then removed in sequential steps, but preserving as much as possible the original surface envelope of the bead model. The method has been fully validated and reported in the literature (Rai et al., Structure 13:723-734, 2005), and a first important application to the structure and conformational changes of integrins has been recently published (Rocco et al., Structure, 2008). Among the main advantages of this method over shell-modelling and grid-based procedures are a better treatment of the hydration water and the preservation of a direct correspondence between beads and original residues. For instance, the latter feature could be used to include flexibility effects into the computations. Furthermore, by identifying and excluding from the hydrodynamic computations beads that are buried and thus non in contact with the solvent, a large span in the size of the structures that can be analysed with this method without loss of precision is obtained: currently, structures from 5K to 250K have been successfully studied.

Modify Lookup Tables:

In this section you can define the hybridizations, atoms and residues that need to be interpreted as beads in the bead model calculation. These parameters are collected in different tables that are used as the components from which the bead sizes are calculated. PDB structures can then be converted to bead models based on the bead parameters defined here.
  • Add/Edit Hybridization: Use this function to modify the hybridization table. In this file you need to list all hybridizations that will be used for the definitions of all atoms. Each hybridization requires a name, a molecular weight and a radius (in Angstrom units). Load the hybridization file first if one exists, and add new hybridizations to it by filling out the fields and clicking on "Add Hybridization to File"
  • Add/Edit Atom: Use this function to modify the atom table. In this file you need to list all atom details that will be used for the definitions of all residues. Each atom requires a name, a hybridization, a molecular weight and a radius (in Angstrom units). In addition, you need to assign a the position of the atom in the molecule. The options are:
    1. Undefined
    2. Main Chain
    3. Side Chain
    4. Prosthetic Group
    5. Other
    Load the hybridization file first, and the atom file if one exists, and add new atoms to it by filling out the fields and clicking on Add Atom to File. You can also delete an atom from the file.
  • Add/Edit Residue: Use this function to modify the residue table. The use of this program is described in this help file.

Modify SOMO Options:

In this section you can set all options for different steps in the program. These options are saved in a system wide config file 
$ULTRASCAN/etc/somo.config
. Every time you close the SOMO program, the currently defined options will be saved in 
$HOME/ultrascan/etc/somo.config
where they will be reloaded from upon startup.
  • ASA Calculation: Use this function to modify the options for the accessible surface area calculation done with SurfRacer (Tsodikov OV, Record MT Jr, Sergeev YV. Novel computer program for fast exact calculation of accessible and molecular surface areas and average surface curvature. J Comput Chem. 2002 23(6):600-9). A detailed description of this module can be found here.
  • Overlap Reduction: This module allows you to change the options for reducing molecular surfaces to beads, and to handle the various issues related to bead overlap elimination. A detailed description of this module can be found here.
  • Hydrodynamic Calculations: This module allows you to change the options for calculating hydrodynamic parameters from the bead models. A detailed description of this module can be found here.
  • Miscellaneous Options: This module allows you to define the volume of hydration of a water molecule and the partials specific volume of your molecule. A detailed description of this module can be found here.
  • Bead Model Output: This module allows you to control output options to be used during the bead model generation. A detailed description of this module can be found here.
  • Reset to Default Options: Click on this button to reset all options to default values. When existing just after clicking on this button will reset the config file as well to default options.

Run SOMO Program:

These functions control the execution of the SOMO program. The first step is to select a lookup table by clicking on Select Lookup Table. You can create multiple lookup tables for different conditions. The lookup table needs to contain all atoms and residues defined in your molecule to be loaded in the next step. As soon as the lookup table has been successfully loaded, the Load PDB File button will become active and you can load a PDB structure file from NMR or X-ray crystallography experiments. The file can contain multiple models and if so, multiple models will be displayed in the list box. You can only process one model at a time. After selecting the model, click on Build Bead Model, which should be active now. This function will generate the beadmodel based on the settings selected above. Alternatively, you can skip all of these steps and load a previously generated beadmodel by clicking on Load Bead Model File from the menu. Once bead model generation is completed, the progress bar will be at 100% and the bead model can be visualized with rasmol by clicking on Visualize Bead Model. rasmol needs to be installed in in 
$ULTRASCAN/bin
for 32 bit machines, and 
$ULTRASCAN/bin64
for 64 bit platforms. You can get a copy of rasmol from http://www.umass.edu/microbio/rasmol/. Once a bead model has been calculated hydrodynamic parameters can be determined from this model by clicking on Calculate Hydrodynamics. The results can be seen as soon as this calculation is complete by clicking on Show Hydrodynamic Calculations.
