SOMO - SAXS/SANS (SAS) Module: MALS+SAXS

Last updated: September 2024

The Help for the MALS+SAXS module, available from the July 2024 intermediate US-SOMO release, is currently still under construction. We apologize for this inconvenience.

This US-SOMO module was conceived to allow joining Multi-Angle Light Scattering (MALS) and Small-Angle X-ray Scattering (SAXS) data collected in sequential or parallel fashion on the same sample, and pre-processed to put them on the same absolute scale (see here and here).

In the image below, the initial screen of the US-SOMO MALS+SAXS module is shown:

As with all other US-SOMO modules, the buttons with the black labels are the ones currently active, and the ones with the red labels are currently inactive but will become active (turning black) when allowed by the processing/visualization stage.

The left side of the window is divided in three sections, labeled "Data files", "Produced Data", and "Messages". By clicking on these labels, the corresponding panel below each label will disappear, allowing for an expansion of the remaining other panel(s). If every panel is made to disappear, the main graph will expand to cover the full size of the MALS+SAXS module window. By clicking again on the labels, the corresponding panels will be restored.

At the top of the left-side panel, you can pre-select a directory where your data are stored. Clicking on this field (shown in the picture above is the /home/ultrascan/ultrascan/somo/saxs path) will open a directory navigator where you can selected another directory. Checking the Lock checkbox will make the selected directory the default one upon re-starting the application.

In the top left panel (Data files) there are five buttons:

  • View Selected, active when up to ten datasets are selected, will show them in text format (see below).

  • The Add files button is used to load either the MALS or the SAXS data. Both type of data should be in either I#(q,t) vs. q or I*(q,t) vs. q (see here for a description of the I#(q) and I*(q) type of data).

    As an example involving data collected in kinetics mode, in the images below we show the loading of fibrin re-polymerization data in which fibrin monomer at 0.8 mg/mL obtained from a thrombin-generated fibrin clot dissolved in CH3COONa 100 mM/CH3COOH 110 mM, pH 4.6, and then subjected to preparative SEC, was mixed in a 1:1 ratio with a TRIS-base 65 mM, NaOH 57 mM, NaCl 100 mM, pH 12.6 buffer, obtaining a final buffer composition of TRIS-base 6 mM/TRIS-NH4+Cl- 265 mM, CH3COONa 105 mM, NaCl 23 mM, pH 7.425 that allows re-polymerization. The rapidly mixed solution was injected sequentially in the MALS and SAXS flow cells (data collected with a dedicated facility at the SWING beamline of the Synchrotron SOLEIL, Gif-sur-Yvette, France).

    The MALS and SAXS original data were first processed with the US-SOMO MALS and HPLC/KIN modules to produce I*(q,t) vs. q datasets on the same, absolute scale (units: [g/mol]).

Below the window with the loaded files listing there is a second section of buttons:

  • Sel. all will select all files.

  • Sel. Unsel. will allow toggling the selection between selected files and everything else not currently selected.

  • Adv. Sel. will open up a panel in which several selection options can be utilized (see here).

  • Average will produce an average with propagated SDs of selected I(q) data. The I(q) values from each frame will be averaged, and then a scaling factor will be determined for each frame against the average resulting frame. The scaling factor for each original frame multiplies the frames's SD. The average intensity SDs are computed as the square root of the sum of the squares of each curves scaled SDs, and this is divided by the number of curves. The resulting datset filename will contain the number of frames averaged, and the initial and final frame numbers, followed by "_avg".

  • Bin allows averaging adjacent points in I(q) datasets, starting with the first point in the file and using a binning size defined in a pop-up dialogue:

  • Smooth performs a regularization of selected data using a moving window, whose dimension is defined in a pop-up menu (shown below), using a Gaussian smoothing kernel of 2n+1 points.

  • I#,I*(q) common times. Pressing this button will launch the first step in the MALS+SAXS datasets joining operation. Especially in kinetic mode, SAXS frames could be collected with longer time intervals in respect to MALS data points, because of both radiation damage issues, and limited sample quantities. In fact, in kinetic mode, while the sample is isolated in the MALS cell and can be continuously illuminated with the relatively low-intensity laser light beam, the sample for the SAXS capillary must be continuosly changed to avoid radiation damage, by pushing it in small volume increments from a sample accumulation loop containing a finite amount of material.
    In addition, the absolute times in the MALS and SAXS datasets can be slightly different. For these reasons, this "common times" utility will use the SAXS frames times to select corresponding times in the MALS datasets, interpolating them from the two closest time points (above and below) corresponding to each SAXS frame. By default, a linear interpolation method will be used, but quadratic and cubic spline interpolation methods are also available from the Options panel, accessible from the corresponding button placed on the bottom line (see here).

    On pressing I#,I*(q) common times, the two datasets are first analyzed, and a pop-up message will report the findings:

    listing the number of individual files with their starting and ending times, and the q-values covered, followed by the determined common time range. Pressing "Ok" will allow the creation of the MALS new dataset with I*(q,t) vs. q datasets having common times with the corresponding available SAXS datasets.

    Before completion, a second pop-up will appear:

    asking if there are other MALS-derived sets of time-dependent parameters (such, e.g., the weight-average molecular weight Mw or the rms z-average radius of gyration [<Rg2>z]0.5) that might be put on a common time scale with SAXS-derived parameters. Pressing "Yes" a directory browser will open allowing selecting a csv-type file where in separate columns the times and the parameters' values must be present (not shown). Pressing "No" the programs will just complete the operation:

    automatically selecting in the graphics window the newly created MALS I*(q,t) vs. q datasets having the same times of the already selected SAXS datasets. This new set of data has "iLin_common" appended to their filenames, and they will also appear in the Produced Data section.

  • Pressing Scroll will allow to examine in the graphics window each single time point consisting of the common MALS and SAXS datasets:

    starting from the first time point. Here we have changed the representation from dots connect by a line to symbols with their associated error bars by selecting the Err checkbox in the Options submenu on top of the graphics windows (see here). The names of the joined MALS+SAXS frames are reported below the cyan-scale bar-wheel, and the time being examined is reported to the right of the bar-wheel.
    Scrolling is performed by either using the cyan-scale bar-wheel, or by clicking on the the "<" and ">" buttons placed at its sides, and can be used to check how the select common time MALS+SAXS frames behave:

Before describing the last two buttons in this panel (Scale and Join I#,I*(q) by time), let's first describe other panels and functions.

The file names of data produced in this module are shown in the Produced Data panel to the centre-left, and can be selected and saved to files using the appropriate buttons below it.

By clicking on the file path shown below the Produced Data label, a file system navigator will open, allowing selecting a different directory and/or creating a new one.

Several buttons are available in this panel:

  • Select all will select all files in this panel (single files can be selected by individually clicking on them, shift-click will select a contiguous series, ctrl-click allows multiple irregularly spaced selections).

  • Sel. Unsel. will allow toggling the selection between selected files and everything else not currently selected.

  • Remove will discard the selected files. If the files were produced by the module, and were not previously saved, a warning window will pop-up, allowing to save them, proceed without saving, or to stop removing the selected items:

  • Show will add the selected file(s) among those produced to the ones already displayed in the graphics window.

  • Show only will show only the selected file(s) among those produced in the graphics window.

Two types of files can be produced:

  • csv-style (Save CSV)

  • or regular 3-columns .dat files (Save).

On the top of the graphics window there is a row of round "Plot buttons:" checkboxes each one controlling the showing of a series of buttons:

The Options checkbox will enable these buttons:

  • Rescale X-Y: adjusts the X-Y axes on the graphics window to maximize the display of shown datasets (no effect on the data themselves).

  • Rescale Y: adjusts only the Y axis on the graphics window to maximize the display of shown datasets only along this direction (no effect on the data themselves).

  • The Log X (Lin X) and Log Y (Lin Y) buttons allow to toggle between linear and log10 scaling of the data on the x- and y-axes, respectively (if zero or negative values are present, they will be temporarily removed when the scale is set to log10 mode, as they cannot be shown on the display in this mode). The buttons will change their respective label once pressed, to underscore what is the action currently available.

  • Selecting the Err checkbox, active when up to 10 files are selected, will switch their representation from the dots connected with a line mode, to symbols (diamonds) with their associated SDs represented as error bars mode.

  • Selecting the Dots checkbox will switch the data representation from the dots connected with a line mode to dots only mode.

  • Each time the Width button is pressed, it increments the data line (or symbol) size of the plots, until after five iterations it goes back to the initial value.

  • Color shifts the colors used in the graphics window; the operation can be repeated until a better contrast with the background is achieved. Note that the background color can be changed by right-clicking on the plot borders, which will open up a pop-up dialogue panel where all plot characteristics can be modified.

  • Legend will turn on below the graphics window a display of the correspondence between colours and filenames (automatically disabled if the selected files are 20 or more and/or if the Err checkbox is selected).

The Selections checkbox will enable these buttons:

  • Select Visible will select the files shown in the graphics window, which can be zoomed using the mouse (left click and drag). For instance, this is a practical way of selecting only a few files, by zooming on a region where only them are present.

  • Remove Vis(ible) will instead remove the files shown in the graphics window.

  • Movie: Pressing this button will open a pop-up window with the commands allowing to view in the main graphics window of the US-SOMO MALS+SAXS module a series of selected data files in a movie-like manner, and to optionally save each frame as an image for real movie-making operations (see here).

  • Save plots allows saving the data shown in any of the plots currently visualized in csv-formatted files. Pressing it will open a pop-up dialogue window where the location and the root filename for the cvs files can be set.


The Cropping checkbox will enable these buttons:

When a part of the graph is selected using the mouse/left button, the buttons become all available (only Crop Zeros and Crop Common are available when files are just displayed after selection).

  • Crop Common will crop all selected files so that they have identical x-axis values by dropping points outside of the union of all selected file's x-axis values. Grid points whose x-axis values differ only by last-digit rounding (e.g., 0.00547318 vs. 0.00547317) are treated as the same point, so curves saved at slightly different rounding do not develop spurious gaps ("holes") in the common grid.

  • Crop Vis(ible) will remove what is shown in the graphics window. For instance, this is practical way to remove "bad" points. As an example, we can remove the lowermost q-values that results from capillary fouling after a certain reaction time from the SAXS datasets, as shown in the image below, before scaling them with the common times MALS datasets. Note: crop visible is not allowed in the middle of a dataset, only at the beginning or at the end!

    We beging by zooming on the region where we want to remove the "bad" datapoints:

    By pressing Crop Vis the shown datapoints will be removed from each SAXS file, as can be seen by zooming out and showing the data as points connected by lines:

  • Crop to Vis(ible) will keep only what is shown in the graphics window. For instance, this is practical way to trim the data on the x-axis.

  • Crop Zeros will remove data having zero or negative values in the intensity columns (discouraged: a warning panel will pop-up, asking the user whether she/he really wants to proceed, since experimental zero or negative values have statistical significance).

The utility works by selecting with the mouse a zone containing a q-values range, and then operating with the provided buttons.

to describe the use of the two last buttons.

  • Scale. On pressing this button, the first time point of the previously selected SAXS datasets and MALS datasets with common times is displayed in the graphics window, and a new series of fields and buttons are shown:

    Here, the Err checkbox has been selected to show the associated error bars.
    Two sets of dashed vertical lines are now displayed, with associated fields having their background color matching the vertical dashed lines colors. On the left side, the pink vertical dashed lines mark the beginning ("Start") and the end ("End") of a zone in the MALS data to be used in the scaling procedure againts the corresponding SAXS data. The corresponding fields are indicated by the "Fit range 1 (scales)" label; "scales" signifies that is this dataset that will be scaled against the other dataset (but at the end of the procedure the scaling can be reversed, see below). On the right side, the light green vertical dashed lines mark the beginning (Start") and the end ("End") of a zone in the SAXS data against which the corresponding MALS data will be scaled. The corresponding fields are indicated by the "Fit range 2 (fixed)" label.

    We beging by adjusting these ranges, by either typing new q-values in the provided fields, or, after having clicked in the corresponding field, by using the cyan-scale bar-wheel, or clicking on the "<" and ">" buttons placed at its sides. The currently shown and used common time point is indicated in the field to the right of the cyan-scale bar-wheel, and can be changed by clicking on this field and using the bar-wheel or the "<" and ">" buttons placed at its sides.
    Normally, the range should be restricted to the 5-6 highest MALS q-values, and to the ∼20 lower q-values SAXS points (sometimes excluding the very noisy intial q-values, which in the example shown are been already excluded as indicated on the top of the graph if the q exclude checkbox is checked, and in any case by the red crosses on the plot):

    Below the ranges fields there are two additional fields and four new buttons.

  • The first field Global scale: indicates the current vertical scaling factor. It is updated at the end of a scaling procedure, but a value can by also typed in it. For instance, this can be useful if an optimal scaling factor has been determined on a restricted set of MALS and SAXS datasets, and then applied to the remaining datasets.

  • MALS SD Mult.: In this field the MALS data standard deviation (or any other form of associated error bars) can be all multiplied by a factor. This option is introduced because usually the MALS-obtained data have associated SD much lower than the corresponding SAXS-obtained data, and thus they will dominate the scaling procedure if a weighted fitting is applied. Moreover, if subsequently the joined MALS+SAXS data will be analyzed by any type of weighted fitting functions, the same problem of over-weighting the MALS data will be present: the multiplying factor entered here can be used to make permanent this change in the global MALS+SAXS resulting datasets. For this example, we will enter a value of 30 for this factor.

  • Fit will perform a trial fit on the currently selected MALS+SAXS common time dataset. Below this line of fields and buttons, the Fit controls are present:

    • First, a list of provided polynomials can be accessed by clicking on the currently selected polynomial:

      with a 3rd degree polynomial being the default option.

    • A series of available fitting procedures can by selected by clicking on the currently selected procedure acronym next to the polynomials section:

      with the LR (Linear Regression) method being the default choice. A description of the other methods available can be found here (to be added).

    • A series of weighting procedures to be applied in the fitting is also provided:

      including no weighting, 1/amount (that is, the inverse of each data point), 1/amount^2 (that is, the inverse of the square of each data point), 1/SD (the default choice), and 1/SD^2.

  • Pressing Fit will show the current fit as a red line between the selected q-ranges:

    This test fit could be also applied to other MALS+SAXS common time datasets, to verify the appropriateness of all the chosen fitting parameters.

Pressing "Cancel" will abort the operation, pressing "Ok" will launch it. On the graphics window, the steps of the global fitting procedure overlaid to the cureently selected MALS+SAXS common time dataset will be shown, followed by the final global vertical scaling results with its value now printed in the Global scale: field:
