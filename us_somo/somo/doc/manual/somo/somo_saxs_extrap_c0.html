Extrapolation to Zero Concentration:

Last updated: July 2026

Synopsis

Small-angle scattering intensities measured at finite concentration contain a concentration-dependent contribution from interparticle interactions (the structure factor). The Extrapolate to zero concentration function removes this contribution by combining a set of I(q) curves measured at different concentrations and extrapolating, at each q, to the ideal dilute limit c = 0.

The function is reached from the SOMO SAXS/SANS Load, manage, and process SAS data module: load or plot the curves of interest, press Load Plotted, select at least three curves, and choose Extrap. to zero conc → Extrapolate. A dialog then opens in which concentrations are assigned and the extrapolation is performed.

The concentration and curve-selection dialog

The dialog lists each selected curve with its concentration. Concentrations are pre-filled from the curve headers (the Conc: tag) or from any value already known to SOMO, and may be edited; each must be a non-negative number. A concentration shown on a red background is not yet a valid number.

Selecting the curves. The rows highlighted (selected) in the table are the curves that will be used. All rows are selected by default. Use the mouse (click, Shift-click for a range, Ctrl-click to toggle individual rows) to change the selection. The Extrapolate button becomes active only when at least three rows are selected and every selected row has a valid concentration; the status line below the table reports the current state.

Note: because the concentration cells remain editable, double-clicking a cell to edit it may collapse the selection to that single row. Set the concentrations first, then choose the selection.

Extrapolation modes

Two modes are offered, selected with the Primus mode checkbox:

Zimm mode (checkbox unchecked, the default). At each q the concentration-normalized intensity I(q,c)/c is fitted linearly against c and the c = 0 intercept is reported. This is the small-angle-scattering analogue of a Zimm plot. The output is a concentration-normalized curve (intensity per unit concentration); it is tagged Conc:1, SOMO's convention for an already-normalized curve, and carries the standard error of the fitted intercept as its uncertainty.

Primus mode (checkbox checked). This reproduces the behavior of the ATSAS almerge program. The highest-concentration curve is taken as a reference; every other curve is scaled onto it by a least-squares factor, and the scaled absolute intensity is then extrapolated linearly against c to c = 0. The output is therefore on the reference curve's absolute intensity scale (a factor of roughly the reference concentration larger than the Zimm result), it inherits the reference curve's error bars, and it is tagged with the reference concentration. Letting the data — rather than the entered concentrations alone — set the relative scale between curves is useful when the curves are different samples or peaks rather than a single dilution series.

Per-q regression plots

Checking Show per-q regression plots in the concentration dialog opens a scrollable pop-up after the extrapolation. It shows, one q value at a time, the linear regression behind the result: each input curve plotted as a point at its concentration (with its error bar), the fitted line, and the extrapolated c = 0 intercept (highlighted, with its own error bar) at the dashed vertical axis. Use the wheel — or the Prev/Next buttons — to step through the q grid; the current q, intercept ± error, and slope are shown below the plot. The y axis matches the mode: I(q)/c for Zimm, scaled absolute I(q) for Primus, or I*(q) for already-normalized inputs. This is a diagnostic view; it does not add or change any plotted curve.

Output

A new curve is added to the plot. Its name is the longest common prefix of the selected curve names followed by _extrap_c0 and a method token: _zimm or _primus (for example Sub_..._extrap_c0_primus). If a curve of that name already exists a numeric suffix is appended. Curves with a concentration of zero are excluded from the fit, and any q point with fewer than two usable values is skipped; the editor window reports how many curves and points were used, and, in Primus mode, the reference curve and the per-curve scale factors.

I*(q) (already-normalized) curves

Curves whose name contains _Istarq_ are treated as already being in I*(q) form (header units g/mol, I(0) = MW), i.e. their intensity has already been normalized by concentration — even though their Conc: header carries the real, non-unity measurement concentration. For these curves the intensity is used as is (it is not divided by concentration again), while the real concentration is still used as the extrapolation variable. A selection made entirely of I*(q) curves therefore produces an already-normalized result (tagged Conc:1 in both modes). Mixing I*(q) curves with raw-intensity curves in one extrapolation is flagged with a warning, as the two are on different scales.

References

The linear extrapolation of concentration-normalized intensity to infinite dilution (Zimm mode) follows the approach introduced in: Zimm, B. H. (1948) The scattering of light and the radial distribution function of high polymer solutions. J. Chem. Phys. 16, 1093-1099.

The concentration extrapolation and the almerge algorithm reproduced by Primus mode are described in: Franke, D., Petoukhov, M. V., Konarev, P. V., Panjkovich, A., Tuukkanen, A., Mertens, H. D. T., Kikhney, A. G., Hajizadeh, N. R., Franklin, J. M., Jeffries, C. M. & Svergun, D. I. (2017) ATSAS 2.8: a comprehensive data analysis suite for small-angle scattering from macromolecular solutions. J. Appl. Cryst. 50, 1212-1225.
