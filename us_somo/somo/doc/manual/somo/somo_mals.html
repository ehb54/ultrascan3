SOMO - SAXS/SANS (SAS) Module: MALS

Last updated: February 2026

The Help for the MALS module, available from the July 2024 intermediate US-SOMO release, is currently still under construction. We apologize for any inconvenience this might cause.

This US-SOMO module was conceived to process Multi-Angle Light Scattering (MALS) data collected either autonomously or sequentially/in parallel with Small-Angle X-ray Scattering (SAXS) data. It is based on the data format produced by a commercial developer of such instruments (Wyatt Technology, now part of Waters Corporation), collected and processed using their ASTRA© software versions 6.1.7.17 and 7.3.2.19, respectively. In the end, the data are saved as a comma-separated variables (csv) file containing in columns first the time (usually [min]) and then the Rayleigh ratios [cm-1] and their uncertainties for each scattering detector, and in lines their time-evolution, with the first line reporting the content of each column, such as in this example (uploaded in ExcelTM):

If available, in a separate block within the same csv file, concentration detector data as a function of time, such as the absorbance at a specific wavelength, can also be stored. This is particularly useful when dealing with chromatographic MALS data (see below).

The data used to develop this module were collected using a Wyatt's now discontinued DAWN HELEOS-II MALS photometer, equipped with a glass flow cell. The geometry of this cell (maintained in their new DAWN model) is based on a solid cylinder on which a narrow channel is bored across the circular face, as exemplified in the schematic drawing below (taken from Bernocco et al. Biophys. J. 79(1) 561-583, 2000). As a consequence, the true scattering angle seen by each detector, placed at fixed positions around the circular face, depend on the refractive index of the solvent:

Therefore, the user has to prepare a text file containing all the necessary information about the detectors available and their effective scattering angle, such as in this example:

which was derived from the "Detailed Report" available in ASTRA© (7.3.2.19). Here the first four lines report the content of the columns that follow: "Detector" (with its physical angle), "Refractive Index Corrected Scattering Angle", "Gain" of each photodiode, and "Normalization Coefficient" of each detector, which is the result of a crucial procedure performed within ASTRA© to compensate for the different physical characteristics of each photodiode and the geometry of their scattering angle. The latter two sets of parameters are not used in our MALS data treatment, since they were used by ASTRA© in producing the Rayleigh ratios at each scattering angle.
Note also that, since one of the photodiodes in the DAWN HELEOS-II MALS photometer utilized for this module development was replaced by a fiber optics for Quasi-Elastic Light Scattering (QELS) (also known as "Dynamic Ligth Scattering (DLS)), in the scattering angles file its position (detector 14 in this case) is marked with a "n/a (0.000)" scattering angle. The US-SOMO MALS module will just avoid processing data from that detector, which, although meaningless, are nevertheless present in the Rayleigh ratios csv exported from ASTRA©.
Data produced by other MALS photometers should be converted to this two-files format and units to be processed by the US-SOMO MALS module.

  • Make I#(t). This function was developed because of the new US-SOMO SAS module features allowing to join Multi-Angle Light Scattering (MALS) data with SAXS data, for global analyses on an expanded q range. Joining is possible only if the intensities of the two datasets are in the same, absolute scale. A first absolute scale for the intensity is in [g2/(cm3*mol)], that we identify with the symbol "I#(t)" or "I#(q)". Later on, by dividing I# data by the concentration of the sample in g/mL, one obtains the data in the absolute scale of [g/mol], which we previously identified with the symbol "I*" (see here for a listing of the required parameters and the equations used to process SAXS data). The I*(q) units are also needed by the Guinier analysis feature of the US-SOMO SAS module, allowing to directly determine the weight-average molecular weight <M>w from the Guinier fit intercept.

The S.D. Util checkbox has two buttons:

This utility, derived from the way the SD are evaluated in the ASTRA© software, estimates the errors associated with the MALS data. This might become useful if no errors have been already associated with the experimental data, or if their reliability is questionable.
The method is based on the assumption that the fluctuations of the signal at the baseline level are a good representation of the error associated with the data at any other point along each I#(t) vs. t chromatogram. Therefore, by estimating the average fluctuations in flat regions of the chromatogram, a constant SD value can be associated with every datapoint in that particular chromatogram. Obviously, chromatograms corresponding to different scattering angles will have different values for their respective constant SD.
Pressing SD Eval will change the graphics window and on the left side of the graph two vertical dashed red lines will appear, which can be repositioned by clicking on the red-background boxes at bottom and entering a different value or using with the mouse the cyan-shades bar-wheel positioned below the graph:

Once an initial set of fitted Gaussians has been produced, it should be globally fitted to all chromatograms. While in the case of SAXS data performing this operation directly on all chromatograms can be very computationally intensive, this is not the case for MALS data, usually containing no more than 20 q-points. Therefore, we will select all the I#(t) vs. t data for the 16 q-values in this FG SEC-MALS example. Importantly, in the global fitting procedure the centers and widths of each particular Gaussian are optimized so to be the same across all chromatograms, and only the amplitudes are then fitted.
